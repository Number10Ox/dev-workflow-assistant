# Phase 7.1: Installation Simplification

## Goal

Reduce DWA installation from 5+ manual steps to a single command with interactive setup wizard.

## Problem Statement

Current installation requires:
1. `npx dwa --install`
2. Install linear-tracker-provider VS Code extension manually
3. Configure Linear API key in settings
4. Install gworkspace-provider VS Code extension manually
5. Configure Google OAuth credentials
6. Learn which commands need which extensions

This friction prevents adoption and makes demos difficult.

## Design Approach

### CLI-First Mode

Core DWA functionality works without any VS Code extensions:

```bash
npx dwa --install          # Works immediately
dwa scaffold my-feature    # No extensions needed
dwa parse                  # No extensions needed
dwa packet DEL-001         # No extensions needed
```

Advanced features gracefully degrade with clear instructions:

```bash
dwa sync-linear
# Error: Linear integration requires setup.
# Run: dwa setup --linear
```

### Interactive Setup Wizard

```bash
npx dwa setup

┌─────────────────────────────────────────────────┐
│           DWA Setup Wizard                      │
└─────────────────────────────────────────────────┘

? What features do you want to enable?
  ◉ Core workflow (scaffold, parse, packets)
  ◯ Linear integration (sync deliverables to tickets)
  ◯ Google Docs import (import specs from Docs)

[Installing core features...]
✓ Skills copied to ~/.claude/skills/
✓ Templates installed
✓ Ready to use!

Next steps:
  cd your-project
  dwa scaffold "My Feature"
```

### Feature Detection

Commands check for required providers at runtime:

```javascript
// In sync-linear.js
async function syncLinear(options) {
  const bridge = new BridgeClient();

  try {
    await bridge.initialize();
  } catch (err) {
    return {
      success: false,
      message: 'Linear integration not configured.\n' +
               'Run: dwa setup --linear\n' +
               'Or install the linear-tracker-provider extension manually.'
    };
  }

  // ... proceed with sync
}
```

## Technical Decisions

### Use `enquirer` for Prompts

[Enquirer](https://www.npmjs.com/package/enquirer) is lightweight and supports:
- Multi-select for feature selection
- Confirm prompts
- Password input (for API keys)
- Works in both terminal and VS Code integrated terminal

### VS Code Extension Installation via CLI

```bash
code --install-extension jedwards.linear-tracker-provider
```

The setup wizard can invoke this directly if VS Code is detected.

### Configuration Storage

Store setup choices in `~/.dwa/config.json`:

```json
{
  "features": {
    "linear": false,
    "googleDocs": false
  },
  "linear": {
    "configured": false
  },
  "googleDocs": {
    "configured": false
  }
}
```

This allows `dwa status` to show what's configured.

## Success Criteria

1. `npx dwa --install` completes in < 10 seconds with zero prompts
2. `npx dwa setup` provides guided configuration for optional features
3. Commands that need missing features show clear setup instructions
4. Demo flow: install → scaffold → parse → packet in under 60 seconds
5. No breaking changes to existing installations

## Out of Scope

- Auto-installing VS Code itself
- Non-VS Code editor support
- CI/CD mode (headless installation)
