<task type="auto">
  <name>Task 2: Create shared utility modules (paths and schema)</name>
  <files>src/utils/paths.js, src/utils/schema.js</files>
  <action>
Create src/utils/paths.js:
- Import os and path from node:os and node:path
- Export getInstallDir(): returns path.join(os.homedir(), '.claude', 'dwa')
- Export getSkillsDir(): returns path.join(os.homedir(), '.claude', 'skills')
- Export getVersionFilePath(): returns path.join(getInstallDir(), '.dwa-version')
- All functions use path.join() exclusively (never string concatenation)

Create src/utils/schema.js:
- Export SCHEMA_VERSION = '1.0.0' (semver string constant)
- Export async function writeJsonWithSchema(filePath, data):
  - Uses write-file-atomic
  - Prepends { schemaVersion: SCHEMA_VERSION } to data object
  - Writes JSON.stringify(payload, null, 2) with utf8 encoding
  - Returns undefined (void) on success, throws on failure
- This is the ONLY way .dwa/ JSON files should be written (enforces REQ-011)
  </action>
  <verify>
- `node -e "const p = require('./src/utils/paths'); console.log(p.getInstallDir())"` outputs path ending in `.claude/dwa`
- `node -e "const p = require('./src/utils/paths'); console.log(p.getSkillsDir())"` outputs path ending in `.claude/skills`
- `node -e "const s = require('./src/utils/schema'); console.log(s.SCHEMA_VERSION)"` outputs "1.0.0"
  </verify>
  <done>paths.js exports three path functions using os.homedir() + path.join(); schema.js exports SCHEMA_VERSION constant and writeJsonWithSchema helper</done>
</task>

<task type="auto">
  <name>Task 3: Create Commander.js CLI program skeleton</name>
  <files>src/cli.js</files>
  <action>
Create src/cli.js:
- Require commander (const { program } = require('commander'))
- Require package.json for version
- Configure program:
  - .name('dwa')
  - .description('Dev Workflow Meta-Framework - deliverable-driven skills for Claude Code')
  - .version(packageJson.version)
- Add three options (NOT subcommands - these are flags):
  - .option('--install', 'Install DWA skills, templates, and references')
  - .option('--upgrade', 'Upgrade existing DWA installation')
  - .option('--uninstall', 'Remove DWA installation completely')
- Parse process.argv
- After parse, check which option is set and call corresponding command handler:
  - If opts.install: require('./commands/install').install()
  - If opts.upgrade: require('./commands/upgrade').upgrade()
  - If opts.uninstall: require('./commands/uninstall').uninstall()
  - If no option: program.help() (show usage)
- Handle mutual exclusivity: if more than one flag passed, show error "Only one operation allowed at a time"
- Do NOT create the command files yet (they'll be in Plan 02 and 03) - use try/catch with helpful error message if command module not found

Export program for testing: module.exports = { program }
  </action>
  <verify>
- `node bin/dwa.js --help` shows all three options (--install, --upgrade, --uninstall)
- `node bin/dwa.js --version` outputs "1.0.0"
- `node bin/dwa.js` (no args) shows help text
- `node bin/dwa.js --install --upgrade` shows mutual exclusivity error
  </verify>
  <done>CLI skeleton accepts --install, --upgrade, --uninstall flags with mutual exclusivity check; --help shows usage; delegates to command modules</done>
</task>
