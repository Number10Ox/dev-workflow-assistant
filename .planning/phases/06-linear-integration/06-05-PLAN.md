  <action>
Add the async routing logic for --sync-linear. Since syncLinear is async but the existing handlers are sync, use an async IIFE wrapper for the sync-linear block only.

1. Add the sync-linear routing block after the uninstall block (after line 60, before the else):
```javascript
} else if (opts.syncLinear) {
  (async () => {
    try {
      const { syncLinear } = require('./commands/sync-linear');
      const result = await syncLinear({
        projectRoot: process.cwd(),
        dryRun: opts.dryRun,
        force: opts.force,
        deliverables: opts.deliverables,
        project: opts.project
      });
      if (!result.success) {
        console.error(result.message);
        process.exit(1);
      }
      console.log(result.message);
    } catch (err) {
      if (err.code === 'MODULE_NOT_FOUND') {
        console.error('Error: sync-linear command not available');
        process.exit(1);
      }
      console.error('Error:', err.message);
      process.exit(1);
    }
  })();
```

2. The async IIFE pattern `(async () => { ... })()` is chosen because:
   - It isolates the async handling to just the sync-linear block
   - Existing sync handlers (install, upgrade, uninstall) remain unchanged
   - Error handling stays within the block with process.exit(1)
   - No changes required to the overall CLI structure

3. Keep MODULE_NOT_FOUND check consistent with existing patterns for graceful degradation.
