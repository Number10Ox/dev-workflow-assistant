  <files>src/google-docs/import.js, test/google-docs/import.test.js</files>
  <action>
1. Create `src/google-docs/import.js`:

   - `parseDocIdFromUrl(input)`:
     - If input matches Google Docs URL pattern, extract document ID
     - URL patterns: `docs.google.com/document/d/{id}/...`
     - If input looks like ID (no slashes, reasonable length), return as-is
     - Throw error for invalid format

   - `wrapWithMarkers(content, docId, revisionId, importHash)`:
     - Returns string with DWA markers per CONTEXT.md:
       ```
       <!-- DWA:SOURCE doc="gdoc" docId="{docId}" -->
       <!-- DWA:IMPORT_BEGIN docId="{docId}" revisionId="{revisionId}" importHash="{importHash}" -->
       {content}
       <!-- DWA:IMPORT_END -->
       ```

   - `importGoogleDoc(options)`:
     - options: { docIdOrUrl, projectRoot, out, force, dryRun }
     - Returns: { success, message, outputPath, report, diagnostics, diffPath? }

     Steps:
     a. Initialize bridge client with GoogleWorkspaceBridgeClient
     b. Check provider availability with checkAvailability()
        - If unavailable, return fatal error with setup instructions
     c. Parse docId from input (URL or ID)
     d. Read document via bridge client (returns raw Google Docs JSON with etag/modifiedTime)
     e. Convert: gdocToMdast -> mdastToMarkdown
     f. Determine output path (explicit resolution):
        - If --out provided, use that path
        - Else, find feature root (walk up looking for .dwa/)
        - If feature root found, use `feature-spec.md` in that directory
        - If NO feature root found, throw error requiring --out:
          "Cannot determine output path. No .dwa/ directory found. Use --out to specify output path, or run from within a DWA feature directory."
     g. Check existing file:
        - If exists, parse markers and extract stored hash
        - Compare with new hash using verifyContentHash()
        - Check document change (prefer etag, else modifiedTime, else revisionId)
        - If unchanged AND document unchanged: skip with "Already at latest"
        - If hash mismatch (local edits detected):
          - Generate diff artifact via generateImportDiff()
          - If --force: overwrite and continue
          - Else: return error with diff path and instructions
     h. Wrap content with markers
     i. Write file (atomic via write-file-atomic)
     j. Generate and write import report
     k. Return result with diagnostics summary and diffPath if generated

   - `generateImportDiff(docId, existingContent, newContent, projectRoot)`:
     - Creates unified diff between existing and new content
     - Writes to .dwa/import-diffs/{docId}/{timestamp}.md
     - Format:
       ```markdown
       # Import Diff: {docId}

       **Generated:** {ISO timestamp}
       **Old hash:** {sha256 of existing}
       **New hash:** {sha256 of new}

       ## Changes

       \`\`\`diff
       {unified diff}
       \`\`\`
       ```
     - Returns path to written diff file
