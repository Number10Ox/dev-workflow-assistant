2. Create `src/google-docs/gdoc-to-mdast.js`:
   - `gdocToMdast(gdocDocument, options = {})` - Main converter
     - Returns { mdast: tree, diagnostics: ImportDiagnostic[], footnoteDefinitions: [] }
     - options.tableMode: 'gfm' | 'html-fallback' (default: 'html-fallback' per CONTEXT.md)
     - Traverses body.content array of StructuralElements
     - Appends footnoteDefinitions at end of root children

   - `convertStructuralElement(element, gdocDocument, context)`:
     - paragraph -> convertParagraph()
     - table -> convertTable()
     - sectionBreak -> { type: 'thematicBreak' }
     - tableOfContents -> null (drop with DWA-GDOC-100 diagnostic)
     - pageBreak -> { type: 'thematicBreak' } + DWA-GDOC-101 diagnostic
     - **Note:** Footnotes and equations are NOT structural elements in real Docs JSON;
       they appear as inline elements within paragraph.elements

   - `convertParagraph(paragraph, gdocDocument, context)`:
     - Check namedStyleType for headings (HEADING_1 through HEADING_6 -> depth 1-6)
     - **List detection via paragraph.bullet (NOT namedStyleType):**
       - If paragraph.bullet exists, this is a list item
       - Use paragraph.bullet.listId to look up list style in document.lists
       - Use paragraph.bullet.nestingLevel for nesting depth
       - Look up document.lists[listId].listProperties.nestingLevels[level] for:
         - glyphType: GLYPH_TYPE_UNSPECIFIED = bullet, DECIMAL/ALPHA = ordered
         - Or use glyphSymbol: "●" = bullet, "☐"/"☑" = checkbox
     - Process elements array (textRun, inlineObjectElement, footnoteReference, equation)
     - Return heading, paragraph, or listItem node

   - `convertInlineElement(element, gdocDocument, context)`:
     - textRun -> convertTextRun()
     - **footnoteReference** -> { type: 'footnoteReference', identifier: `fn${footnoteNumber}` }
       - Collect footnote content from document.footnotes[footnoteId]
       - Add to context.footnoteDefinitions for appending at document end
     - **equation** (inline) -> placeholder or math if extractable
       - Emit DWA-GDOC-204 diagnostic if not extractable
     - inlineObjectElement -> handle image (see below)

   - `convertTextRun(textRun)`:
     - Extract content and textStyle
     - Apply formatting innermost to outermost: strikethrough -> emphasis -> strong
     - Handle inline code (monospace font) -> inlineCode
     - Trim trailing \n from content (Google Docs adds newlines)

   - `analyzeTableComplexity(table)`:
     - **Defensive field access** (actual Docs JSON field locations may vary):
       - Check tableCell.tableCellStyle.rowSpan / columnSpan (if present)
       - OR check tableCell.rowSpan / columnSpan directly
       - Treat missing span fields as 1 (default, no merge)
     - Return 'simple' if all spans = 1
     - Return 'complex' if:
       - Any rowSpan > 1 or columnSpan > 1
       - OR multi-paragraph cells (cell.content.length > 1)
       - OR nested lists within cells (for maximum fidelity)

   - `convertTable(table, gdocDocument, diagnostics, mode)`:
     - If mode='gfm' OR (mode='html-fallback' AND complexity='simple'):
       - Convert to mdast table node with align array
       - Extract alignment from tableCellStyle.contentAlignment
     - If mode='html-fallback' AND complexity='complex':
       - Add DWA-GDOC-202 diagnostic
       - Return { type: 'html', value: '<table>...</table>' }
       - Include colspan/rowspan attributes in HTML output

   - `convertFootnote(footnoteId, gdocDocument, diagnostics)`:
     - Look up footnote content in document.footnotes[footnoteId]
     - Return mdast footnoteReference { type: 'footnoteReference', identifier: 'fn{N}' }
     - Collect footnoteDefinition for appending at end of document

   - `convertEquation(equation, diagnostics)`:
     - If equation has extractable text: emit as `$...$` or `$$...$$`
     - Otherwise: emit HTML placeholder with DWA-GDOC-204 diagnostic
       `<!-- DWA:EQUATION id="{id}" status="unconverted" -->`

   - **mdast Subset Emitted** (per design doc):
     - heading, paragraph, text, strong, emphasis, delete, inlineCode
     - link, table, html (for complex tables), list, listItem
     - thematicBreak, footnoteReference, footnoteDefinition, image
     - Never emitted: random styling spans, font-specific nodes
