2. Create `src/google-docs/gdoc-to-mdast.js`:
   - `gdocToMdast(gdocDocument, options = {})` - Main converter
     - Returns { mdast: tree, diagnostics: ImportDiagnostic[] }
     - options.tableMode: 'gfm' | 'html-fallback' (default: 'html-fallback' per CONTEXT.md)
     - Traverses body.content array of StructuralElements

   - `convertStructuralElement(element, gdocDocument, diagnostics)`:
     - paragraph -> convertParagraph()
     - table -> convertTable()
     - sectionBreak -> { type: 'thematicBreak' }
     - tableOfContents -> null (drop with DWA-GDOC-100 diagnostic)

   - `convertParagraph(paragraph, gdocDocument, diagnostics)`:
     - Check namedStyleType for headings (HEADING_1 through HEADING_6 -> depth 1-6)
     - Process elements array (textRun, inlineObjectElement)
     - Handle lists: bulletedList/numberedList namedStyleType -> list nodes
     - Return heading, paragraph, or listItem node

   - `convertTextRun(textRun)`:
     - Extract content and textStyle
     - Apply formatting innermost to outermost: strikethrough -> emphasis -> strong
     - Handle inline code (monospace font) -> inlineCode
     - Trim trailing \n from content (Google Docs adds newlines)

   - `analyzeTableComplexity(table)`:
     - Return 'simple' if no merged cells (all rowSpan=1, columnSpan=1)
     - Return 'complex' if any merged cells exist

   - `convertTable(table, gdocDocument, diagnostics, mode)`:
     - If mode='gfm' OR (mode='html-fallback' AND complexity='simple'):
       - Convert to mdast table node with align array
       - Extract alignment from tableCellStyle.contentAlignment
     - If mode='html-fallback' AND complexity='complex':
       - Add DWA-GDOC-202 diagnostic
       - Return { type: 'html', value: '<table>...</table>' }
