1. Create `skills/dwa-generate-pr-description/skill.md`:
   ```markdown
   ---
   name: generate-pr-description
   description: Generate PR description from deliverable metadata. Outputs to clipboard or file.
   disable-model-invocation: false
   argument-hint: <DEL-###> [--output clipboard|file|stdout]
   ---

   # DWA: Generate PR Description

   ## Purpose

   Generate a formatted pull request description from deliverable metadata including user story, acceptance criteria, and drift summary. The output is ready to paste into your PR.

   ## Critical Rules

   **MUST NOT:**
   - Modify spec, registry, or any DWA files (READ-ONLY skill)
   - Generate content without a valid deliverable ID
   - Include sensitive information in PR descriptions

   **MUST:**
   - Verify deliverable exists before generating
   - Include drift summary if open drift exists
   - Report output location to user
   - Handle clipboard failure gracefully

   ## Process

   ### Step 1: Parse arguments

   ```javascript
   // Required: deliverable ID (DEL-###)
   // Optional: --output [clipboard|file|stdout] (default: clipboard)

   const deliverableId = args[0]; // e.g., "DEL-001"
   const outputArg = args.find(a => a.startsWith('--output'));
   const output = outputArg?.split('=')[1] || 'clipboard';
   ```

   ### Step 2: Verify deliverable exists

   ```javascript
   const deliverablePath = path.join(process.cwd(), '.dwa', 'deliverables', `${deliverableId}.json`);
   if (!fs.existsSync(deliverablePath)) {
     console.error(`Deliverable ${deliverableId} not found`);
     console.error(`Expected: ${deliverablePath}`);
     return;
   }
   ```

   ### Step 3: Generate PR description

   ```javascript
   const { generatePRDescription } = require('[DWA_INSTALL_PATH]/src/pr-description/generate');

   const result = await generatePRDescription({
     deliverableId,
     projectRoot: process.cwd(),
     output
   });
   ```

   ### Step 4: Report result

   If clipboard:
   - "PR description copied to clipboard"
   - "Paste into your PR on GitHub/GitLab"

   If file:
   - "PR description saved to: {path}"
   - "Open and copy the content to your PR"

   If stdout:
   - Output the content directly
   - "Copy the above content to your PR"

   ### Step 5: Suggest next steps

   - "Review the 'Changes Made' section and fill in implementation details"
   - "Check off acceptance criteria as you verify them"
   - If drift exists: "Review drift items and run `/dwa:propose-drift-patches {id}` if needed"

   ## Output Format

   The generated PR description follows this structure:

   ```markdown
   ## Summary

   **Deliverable:** DEL-001
   **Linear:** [DEL-001](https://linear.app/...)

   ### User Story

   As a developer, I want to import Google Docs so that I can convert specs to canonical format.

   ### Description

   Import feature converts Google Docs to markdown format preserving tables.

   ## Changes Made

   <!-- User fills this in -->

   ## Acceptance Criteria

   - [ ] Import converts paragraphs to markdown
   - [ ] Import converts tables to GFM or HTML

   ## Testing

   **QA Notes:** Test with complex tables and images

   ## Drift Summary (if applicable)

   **Open items:** 2

   - **pending:** Added retry logic for rate limits
   - **accept:** Changed error format to structured object

   ---
   *Generated by DWA at 2026-01-25T10:00:00Z*
   ```

   ## Notes

   - This skill uses LLM capabilities (`disable-model-invocation: false`) for context-aware suggestions
   - PR descriptions are ALWAYS read-only - never modifies source files
   - Clipboard output requires clipboard access; falls back to file if unavailable
   - Run this skill when you're ready to create a PR for a completed deliverable
   ```
