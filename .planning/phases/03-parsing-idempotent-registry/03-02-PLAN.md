<feature>
  <name>Idempotent Registry Updates</name>
  <files>src/parser/registry.js, src/commands/parse.js, tests/registry.test.js, tests/parse-command.test.js</files>
  <behavior>
    Given parsed deliverables from parseSpec and registry directory:

    updateRegistry(deliverables, registryDir):
    - For each deliverable:
      - If DEL-###.json exists: merge spec fields, preserve runtime fields (status, linear_id, pr_url, completed_at)
      - If orphaned flags exist: remove them (un-orphan)
      - Deep compare with existing; skip write if unchanged
      - Write atomically via writeJsonWithSchema
    - For existing files not in parsed set:
      - Add orphaned: true, orphaned_at: timestamp (if not already flagged)
    - Return { created, updated, unchanged, orphaned } counts

    runParse(specPath, projectDir):
    - Call parseSpec(specPath)
    - If errors: return { success: false, errors }
    - Call updateRegistry(deliverables, projectDir/.dwa/deliverables)
    - Return { success: true, summary: { parsed, created, updated, unchanged, orphaned }, warnings }

    Cases:
    - Fresh parse 3 deliverables -> creates 3 files, returns { created: 3 }
    - Re-parse unchanged -> { unchanged: 3, created: 0, updated: 0 }
    - Re-parse with edit to DEL-001 -> { updated: 1, unchanged: 2 }
    - Re-parse missing DEL-002 -> { orphaned: 1, unchanged: 2 }
    - Re-parse with DEL-002 restored -> orphaned flags removed, { updated: 1 }
    - Existing DEL with status, linear_id -> preserved after re-parse
  </behavior>
  <implementation>
    1. Create src/parser/registry.js with:
       - SPEC_FIELDS = ['id', 'user_story', 'description', 'acceptance_criteria', 'qa_notes', 'dependencies']
       - RUNTIME_FIELDS = ['status', 'linear_id', 'linear_url', 'pr_url', 'completed_at', 'created_at']
       - updateRegistry function: iterate deliverables, merge, compare, write
       - Handle orphan flagging for missing IDs
    2. Create src/commands/parse.js with:
       - runParse function: orchestrates parseSpec + updateRegistry
       - Ensures .dwa/deliverables directory exists
       - Returns structured result with counts and any warnings
    3. Tests verify idempotency, field preservation, orphan handling
  </implementation>
</feature>
