<feature>
  <name>Spec Parser with Validation</name>
  <files>src/parser/parse-spec.js, src/parser/validate.js, tests/parse-spec.test.js</files>
  <behavior>
    Given a feature-spec.md file path:
    - Extract YAML frontmatter with gray-matter
    - Parse markdown to AST with remark + remark-parse + remark-gfm
    - Find Deliverables Table (table under "### 3.1 Deliverables Table" heading)
    - Validate frontmatter: feature_id, title, spec_schema_version required
    - Validate table structure: required columns present (Deliverable ID, User Story, Description, Acceptance Criteria, QA Plan Notes)
    - Validate content: IDs unique, required cells non-empty
    - Return { frontMatter, deliverables, errors, warnings }

    Cases:
    - Valid spec with 2 deliverables -> { deliverables: [DEL-001, DEL-002], errors: [] }
    - Missing feature_id -> { errors: [DWA-E010: Missing required field: feature_id] }
    - Duplicate ID -> { errors: [DWA-E031 Row 3: Duplicate Deliverable ID: DEL-001] }
    - No table found -> { errors: [DWA-E020: Deliverables Table not found] }
  </behavior>
  <implementation>
    1. Install remark ecosystem: npm install remark remark-parse remark-gfm unified unist-util-visit fast-deep-equal
    2. Create src/parser/validate.js with ValidationError class and validation functions
    3. Create src/parser/parse-spec.js with parseSpec function using:
       - gray-matter for frontmatter
       - unified().use(remarkParse).use(remarkGfm).parse() for AST
       - visit(tree, 'table', ...) to find table
       - extractCellText helper for nested markdown in cells
    4. Tests verify each validation case + successful parsing
  </implementation>
</feature>
