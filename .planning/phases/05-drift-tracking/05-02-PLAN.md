<feature>
  <name>Complete Deliverable Command (TDD)</name>
  <files>src/commands/complete.js, tests/commands/complete.test.js</files>
  <behavior>
    runComplete(deliverableId, options, projectRoot) orchestrates completion:

    options: {
      prUrl: string (optional),
      commitSha: string (optional),
      notes: string (optional),
      driftDecision: 'accept' | 'revert' | 'escalate' | 'pending' (optional),
      appliesToNext: boolean (optional, default false)
    }

    Returns: {
      success: boolean,
      driftDetected: boolean,
      driftRecorded: boolean,
      errors: Array<{code: string, message: string}>
    }

    Flow:
    1. Validate deliverable exists in registry (DWA-E080 if not)
    2. Run structural drift detection via detectStructuralDrift
    3. If drift detected AND driftDecision provided -> append drift event
    4. If drift detected AND no driftDecision -> set driftDetected: true (caller prompts)
    5. Update registry: status='completed', pr_url, completed_at, last_completed_commit
    6. Return structured result

    Cases:
    - Deliverable not found -> errors: [{code: DWA-E080, message: 'Deliverable DEL-001 not found'}]
    - No drift, simple completion -> success: true, driftDetected: false, driftRecorded: false
    - Drift detected, decision provided -> success: true, driftDetected: true, driftRecorded: true
    - Drift detected, no decision -> success: true, driftDetected: true, driftRecorded: false
    - Invalid drift decision -> errors: [{code: DWA-E072, message: 'Invalid drift decision...'}]
    - PR URL captured -> registry.pr_url updated
    - Commit SHA captured -> registry.last_completed_commit updated
    - Already completed -> re-completes, updates timestamps (idempotent-ish)
  </behavior>
  <implementation>
    Follow start.js pattern for command structure.
    Import appendDriftEvent and detectStructuralDrift from Phase 05-01 utilities.
    Use writeJsonWithSchema for atomic registry updates.
    Drift event recorded with source: 'complete_command'.
    Evidence refs populated from prUrl and commitSha.
  </implementation>
</feature>
